# 2026-05-22 작업 메모

## 오늘 완료한 것

- Telegram 봇을 launchd 백그라운드 실행 방식으로 정리했다.
- `.env` 자동 로드, 재시작 스크립트, 시작 로그 구분선을 추가했다.
- 개인 채팅에서는 누구나 사용 가능하게 바꾸고, 그룹 채팅은 계속 차단하도록 유지했다.
- `/watch`, `/unwatch`, `/watchlist` 관심종목 명령을 추가했다.
- `/company`, `/watch`에서 회사 확인 버튼을 먼저 보여주도록 변경했다.
- `/company`는 Buffett, Graham, Lynch, Munger 중 원하는 관점을 버튼으로 선택하도록 바꿨다.
- 로컬 Ollama 모델을 사용해 선택한 투자 전문가 관점의 해석을 생성하도록 연결했다.
- LLM 분석이 오래 걸릴 때 버튼 클릭 직후 `자료 조사중...` 메시지를 먼저 보여주고, 완료 후 새 메시지로 리포트를 보내도록 개선했다.
- 현재 `.env`에는 `LOCAL_LLM_MODEL=qwen3.6:27b`를 설정했다.

## 현재 사용 흐름

```text
/company AAPL
→ 회사 정보 확인
→ Buffett / Graham / Lynch / Munger 중 선택
→ 자료 조사중...
→ 선택한 전문가 관점 리포트 도착
```

관심종목은 다음처럼 사용한다.

```text
/watch AAPL
/watchlist
/unwatch AAPL
```

## 다음 작업 후보

### 티커 검색

일반 사용자는 티커를 모르는 경우가 많으므로 `/ticker 회사명` 명령을 추가하는 방향이 좋다.

Yahoo Finance 검색은 편하지만 공식성이 약하므로, 다음 구현은 SEC 공식 파일을 1차 소스로 사용한다.

```text
https://www.sec.gov/files/company_tickers_exchange.json
```

예상 흐름:

```text
/ticker apple
→ AAPL / Apple Inc. / Nasdaq / CIK 320193
→ 분석하려면 /company AAPL
→ 관심종목 추가는 /watch AAPL
```

주의할 점:

- SEC는 기업 식별에는 좋지만 ETF/펀드 검색은 약할 수 있다.
- 한글 회사명 검색은 별칭 사전이 없으면 어렵다.
- SEC 파일은 매 요청마다 받지 말고 캐시를 두는 편이 좋다.
- CIK를 저장하면 나중에 SEC 공시/재무 데이터 연동에 유리하다.
